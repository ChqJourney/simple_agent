# work agent 用户指南

这份指南面向普通用户，不要求你会写代码。

`work agent` 是一个桌面 AI 助手。它不只是聊天，还可以在你选定的工作区里：

- 阅读和搜索文档
- 整理文件
- 修改工作区内的文件
- 跟着任务分步执行
- 识别图片或扫描 PDF 里的文字

## 1. 先理解两个概念

### 工作区

工作区就是你选给它的一个本地文件夹。

- Agent 读取和写入本地文件时，默认都围绕这个文件夹进行
- 每个工作区有自己的聊天记录、会话列表和文件树
- 聊天记录会保存到这个工作区里的 `.agent` 目录

### 模型

模型是你在设置里配置的 AI 服务。

- 首次使用前必须先配置
- 你需要提供对应平台的 `API Key`
- 不同模型的能力不同，比如有的支持图片，有的不支持

## 2. 第一次使用

### 第一步：创建工作区

打开应用后，点击创建工作区，选择一个本地文件夹。

建议：

- 给每个项目或文档集合单独建一个工作区
- 不要直接选太大的磁盘根目录

### 第二步：配置模型

进入 `Settings` -> `Model`：

1. 选择主模型 `Primary`
2. 选择 provider
3. 选择 model
4. 输入 `API Key`
5. 如有需要，填写 `Base URL`
6. 点击测试连接
7. 点击底部 `Save`

如果你希望后台任务和主对话分开跑，还可以配置 `Background` 模型。

### 第三步：开始聊天

进入工作区后，在底部输入框输入你的需求，按 `Enter` 发送。

常见例子：

- “帮我总结这个文件夹里的产品文档”
- “搜索所有提到退款政策的文件”
- “把这个 README 改成更适合新人的版本”
- “帮我找出这个项目里所有 hardcoded URL”

## 3. 工作区界面怎么用

### 顶栏

顶栏可能显示：

- 当前工作区名称
- token 用量
- OCR 状态
- WebSocket 状态
- 当前模型
- 运行时间线入口
- 会话压缩状态

如果你看到：

- `Compacting`：系统正在压缩长会话上下文
- `Compacted`：压缩刚刚完成
- `OCR unavailable`：你已启用 OCR，但 OCR sidecar 还没安装或当前不可用

### 左侧面板

左侧主要有四部分：

- 当前工作区信息
- `Skills`
- `Tools`
- 会话列表

你可以在这里：

- 新建会话
- 切换历史会话
- 删除会话
- 查看当前工作区可用工具数量
- 查看系统技能和工作区技能

### 中间聊天区

这里会显示：

- 你的消息
- AI 回复
- 推理过程
- 工具调用
- 工具结果
- 失败重试按钮
- delegated worker 卡片

### 右侧面板

右侧有两个标签：

- `File Tree`
- `Tasks`

`File Tree` 可以：

- 浏览工作区文件
- 打开工作区文件夹
- 导入文件到工作区
- 把文件路径拖进输入框

`Tasks` 会显示 Agent 显式创建的任务清单。  
不是每次聊天都会出现任务。

## 4. 聊天时最常用的操作

### 发送与换行

- `Enter`：发送
- `Shift + Enter`：换行

### 停止生成

当 AI 还在输出时，可以点击停止按钮中断当前回复。

### 复制回复

每条 AI 回复都可以复制。

### 查看详细过程

AI 回复上方会有可折叠的详细过程区域，里面可能包含：

- thinking / reasoning
- tool call
- tool result
- 用户审批记录

如果你只关心最终答案，可以不展开。

## 5. 图片、文件和拖拽

### 图片附件

如果当前模型支持图片输入，你可以：

- 把图片拖进输入区域
- 在输入框里粘贴截图或图片

图片会作为附件发送给模型。

注意：

- 是否支持图片，取决于你当前选的模型
- 如果模型不支持图片，附件功能会受限或报错

### 从文件树拖路径

你可以把右侧文件树里的文件拖到输入框里。

效果：

- 图片文件通常会作为图片附件处理
- 非图片文件会插入成一个高亮路径标签
- 发送时，系统会把它转换成真实路径给 Agent

这很适合说：

- “请检查这个文件：`xxx.md`”
- “对比这两个文件”
- “把这个配置文件里的端口改成 8080”

### 导入文件到工作区

在右侧文件树顶部有导入按钮，可以把外部文件复制进当前工作区。

适合场景：

- 把客户发来的文档放进工作区
- 把截图或 PDF 拉进来再让 Agent 处理

## 6. 审批模式和执行模式

每个会话都有两种执行模式：

### Regular

这是默认模式，更安全。

当 Agent 要做高风险操作时，会先弹确认框，比如：

- 写文件
- 执行 shell 命令
- 执行 Python
- 执行 Node

确认框里有四个选项：

- `Reject`
- `Approve Once`
- `Always This Session`
- `Always This Workspace`

适合：

- 第一次使用
- 处理重要文件
- 不想让 Agent 自动改文件的时候

### Free

这个模式会跳过需要审批的工具确认。

适合：

- 你已经很清楚要它做什么
- 你希望它连续完成多步操作

风险：

- 它可能直接改文件
- 它可能直接执行命令

如果你不确定，优先用 `Regular`。

## 7. Settings 怎么配

### 7.1 Model

这里配置模型。

你可以设置：

- 主模型 `Primary`
- 后台模型 `Background`
- provider
- model
- API Key
- Base URL
- 是否开启 reasoning

建议：

- 日常文本任务：先把主模型配好
- 长会话、标题生成、后台任务多：再配 `Background`

### 7.2 Runtime

这里配置运行参数和额外指令。

包括：

- 上下文长度
- 最大输出 token
- 最大工具轮次
- 最大重试次数
- delegated task 超时
- 自定义 system prompt

如果你不是高级用户，通常保持默认值即可。

### 7.3 Tools

这里可以逐个启用或禁用工具。

适合场景：

- 不希望 Agent 访问网络时，关闭 `web_fetch`
- 不希望 Agent 运行命令时，关闭执行类工具
- 只想让它做文档整理时，保留文档类工具

### 7.4 Skills

这里控制本地技能系统。

你可以：

- 开启或关闭本地 skills provider
- 单独禁用 system skills
- 查看系统技能目录

说明：

- 工作区技能会从当前工作区的 `.agent/skills` 扫描
- 关闭本地 skills provider 后，skills 就不会参与运行时

### 7.5 OCR

这里配置 OCR。

你可以：

- 开启/关闭 OCR 功能
- 查看是否已安装 OCR sidecar
- 安装 OCR sidecar

安装方式不是在线下载，而是：

1. 点击安装按钮
2. 选择本地 OCR sidecar 文件夹
3. 安装到应用目录

适合处理：

- 截图
- 扫描版 PDF
- 图片里的表格或文字

### 7.6 UI

这里可以改：

- 界面语言：中文 / English
- 显示模式：跟随系统 / 亮色 / 暗色
- 基础字号

## 8. OCR 什么时候用

普通文本 PDF 和 Markdown，优先直接让 Agent 读取。  
只有在下面这些情况，再考虑 OCR：

- 图片里有文字
- PDF 是扫描件
- 直接读取 PDF 时内容不完整

你可以直接说：

- “识别这张图片里的文字”
- “这份扫描 PDF 看不清正文，试试 OCR”

## 9. 对普通用户最有用的使用技巧

### 技巧 1：把目标说具体

不要只说：

- “帮我看看这些文件”

更好的是：

- “请先找出这个工作区里和发票相关的文件，再总结处理流程”

### 技巧 2：先找，再读，再改

复杂任务推荐分三步：

1. “先搜索……”
2. “再读取这几个文件……”
3. “最后帮我改成……”

这样更稳定，也更容易检查。

### 技巧 3：善用文件拖拽

如果你已经知道要处理哪个文件，直接把它拖进输入框，比口头描述文件名更准确。

### 技巧 4：先用 Regular，熟悉后再用 Free

刚开始先保持 `Regular`，等你熟悉它的行为，再考虑 `Free`。

### 技巧 5：长会话不用太担心

当上下文变长时，系统会自动做会话压缩。  
你可能会在顶栏看到 `Compacting` 或 `Compacted`，这是正常现象。

## 10. 常见场景示例

### 场景 1：总结一个文件夹

可以说：

“请阅读这个工作区里的文档，按主题总结，并列出最重要的 5 个结论。”

### 场景 2：整理文档

可以说：

“请检查这份 README，找出过时内容，并直接改成适合新人的版本。”

### 场景 3：从截图提取内容

可以说：

“我贴了一张截图，请提取文字并整理成要点。”

### 场景 4：在很多文件里找信息

可以说：

“搜索所有提到合同终止条件的文件，并把命中的位置列出来。”

### 场景 5：让它一步步执行

可以说：

“请先列出任务计划，再逐项执行；如果要改文件，先告诉我会改哪些文件。”

## 11. 隐私与数据说明

使用前最好知道这几点：

- 本地聊天记录会保存在工作区 `.agent/` 下
- 工作区列表和设置会保存在应用本地
- 你的对话内容会发送到你配置的模型服务 provider
- 如果启用了 `web_fetch`，Agent 可以访问公网网页
- 本地文件操作默认围绕当前工作区进行

如果你处理的是敏感数据，建议：

- 使用单独工作区
- 关闭不需要的工具
- 谨慎使用 `Free` 模式

## 12. 遇到问题时先看这里

### 连接测试失败

检查：

- API Key 是否正确
- Base URL 是否正确
- 网络是否正常
- 选的 provider 和 model 是否匹配

### 图片发不上去

通常是当前模型不支持图片输入。  
换一个支持图片的模型再试。

### OCR 不可用

检查：

- OCR 是否已启用
- OCR sidecar 是否已安装
- 安装目录是否完整

### 切换页面时被拦住

如果当前会话还在生成，或者正在做会话压缩，离开工作区时会弹确认框。  
这是为了防止你误中断任务。

### 会话切换时提示停止任务

如果当前会话正在回复，切到另一个会话时会提示是否中断当前回复。  
这是正常行为。

## 13. 最后一个建议

把它当作“会操作文件的助手”，而不是“总能一次成功的黑盒”。

最好的使用方式通常是：

1. 给清晰目标
2. 观察它找到的文件和中间步骤
3. 关键修改前确认
4. 满意后再让它批量执行

这样效率最高，也最安心。
